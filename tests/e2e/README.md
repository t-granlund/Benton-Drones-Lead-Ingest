# E2E Test Suite ‚Äî Lead-Ingest

Autonomous end-to-end tests generated by the E2E Testing Expert. Black-box HTTP
tests against a real instance of the stdlib server on an ephemeral port, each
class backed by its own isolated temp SQLite database (no shared state, no
network, no production credentials).

## Run

```bash
# Dedicated E2E suite (55 tests, ~6s)
python -m unittest discover -s tests/e2e -t tests/e2e -v
```

The suite is **isolated** from `make test` (which still runs the 281 existing
unit/E2E tests unchanged). Each test class spins up its own
`ThreadingHTTPServer` on `127.0.0.1:0` with a fresh temp DB.

## Files

- `e2e_base.py` ‚Äî shared `E2ETestBase` fixture + helpers (not a test module).
- `test_e2e_*.py` ‚Äî test modules.
- `e2e_manifest.json` ‚Äî 1:1 feature ‚Üí source ‚Üí test mapping (source of truth).
- `e2e_report.json` ‚Äî latest execution results.

## Idempotency

Re-running on an unchanged codebase produces zero changes to the test files and
the same 55/55 pass result. Features removed in future are marked `skip`, never
deleted.
