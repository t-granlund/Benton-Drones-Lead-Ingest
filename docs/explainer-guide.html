Lead Ingest Initiative

A self-owned, Shopify-aware lead ingestion pipeline for Benton Drones.

This guide explains the in-house lead ingestion pipeline built for Benton Drones. It covers why the system exists, how it works, who built it, and what must happen before it goes live.

What this replaces

The current workflow uses Google Forms + PDFfiller + Google Sheets + manual Google Earth work. This project replaces that with a self-owned, Shopify-aware lead backend that keeps PII, consent, signatures, and mapping data under Benton Drones control.

Data flow

  1. A visitor arrives from a Shopify CTA or direct link (/signup or /signup/<variant>).
  2. The page captures campaign/source/variant attribution and optional Shopify context.
  3. The visitor reads the waiver, checks agreement, and types their full legal name as an electronic signature.
  4. On POST, the server validates input, checks CSRF and honeypot, geocodes the address, and stores the lead plus consent/signature audit records.
  5. Admins log in at /admin-login to view analytics, a Leaflet map, recent leads, and drill-down details.
  6. Protected exports at /export/csv, /export/geojson, and /export/kml can be downloaded for planning.

Custom tools

No new Universal Constructor tools were required for this phase. The system uses:

  • Python standard library http.server and sqlite3.
  • Vendored Leaflet (CSS + JS) in static/leaflet/ for the admin map.
  • Mock geocoder (lead_ingest/geocoding.py) for local development; production will swap in a real provider.

Run it locally

python scripts/init_db.py
ADMIN_PASSWORD=your-admin-password python -m lead_ingest.server

Then open:

  • http://127.0.0.1:8000/signup
  • http://127.0.0.1:8000/admin-login
  • http://127.0.0.1:8000/admin
  • http://127.0.0.1:8000/overview

Run tests:

python -m unittest discover -s tests

Production deployment path

Blocked Human decisions required:

  1. Choose a host for leads.bentondrones.com (Fly.io, Railway, Render, Hetzner, VPS, etc.).
  2. Migrate DNS authority to Cloudflare and add the leads A/CNAME record.
  3. Swap the mock geocoder for a real provider and add provider API keys as environment secrets.
  4. Replace the placeholder waiver text with legally reviewed language.
  5. Set strong ADMIN_PASSWORD, CSRF_SECRET, and ADMIN_SESSION_SECRET in production.
  6. Configure HTTPS, backups, monitoring, and a persistent database strategy (Postgres/PostGIS recommended).

Human gates

Warning The waiver text is a placeholder.

The current waiver is clearly labeled [PLACEHOLDER WAIVER — MUST BE REVIEWED BY LEGAL COUNSEL BEFORE USE]. Before production, a qualified attorney must review and replace the language in lead_ingest/models.py.

The electronic signature disclaimer also states that typing a full legal name constitutes a legally binding signature. This should be reviewed alongside the waiver.
