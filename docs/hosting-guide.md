# Benton Drones Lead Ingest — Hosting Guide

The source code lives on GitHub at:

```
https://github.com/t-granlund/Benton-Drones-Lead-Ingest
```

This guide covers how to host the static HTML pages and how a friend can clone and run the full system locally on a Mac.

## Static HTML pages that can be hosted

These files are pure HTML/CSS and can be served by any static host:

| File | Purpose |
|---|---|
| `index.html` | Root redirect to the training guide |
| `static/landing-page.html` | Public marketing landing page |
| `docs/explainer-guide.html` | Initiative overview and architecture |
| `docs/architecture-diagram.html` | Visual system architecture diagram |
| `docs/training-guide.html` | User/admin/developer training guide |

## Pages that require the Python backend

These routes are generated by `lead_ingest/server.py` and cannot be hosted as static files:

- `/signup` — the actual signup form
- `/admin-login` and `/admin` — admin dashboard
- `/admin/lead/<id>` — lead detail
- `/admin/lead/<id>/print` and `/admin/lead/<id>/pdf` — consent form exports
- `/export/csv`, `/export/geojson`, `/export/kml` — data exports
- `/overview`, `/shopify-preview`, `/changelog`, etc. — generated project pages

To test these end-to-end, you must run the Python application locally.

### Option 1: GitHub Pages (currently enabled)

**Pros:** free for public repos, integrated with GitHub, automatic deploys on push, HTTPS.  
**Cons:** public repo required (now satisfied), basic CDN.

Pages is already configured with source `main` branch and root `/`.

### Option 2: Vercel

**Why Vercel is also a strong choice:**
- Excellent free hobby plan
- One-click import from GitHub
- Automatic deployments + preview deployments
- Fast global CDN
- Custom domain support

Useful if you want better performance, preview URLs, or may make the repo private later.

### Option 3: Cloudflare Pages

**Why Cloudflare Pages is also excellent:**
- Free plan
- Unlimited bandwidth
- Very fast global CDN
- Custom domain support with free automatic HTTPS
- More control than Vercel

### Trade-offs

- **GitHub Pages** is simplest and already working.
- **Vercel/Cloudflare** are better if you want a custom domain, faster CDN, or might make the repo private later.
- The hosted pages are public. The source code is also public. Do not put secrets or PII in the repo.

## How a friend clones and runs the full system on a Mac

Send them these commands:

```bash
# Clone the repo
git clone https://github.com/t-granlund/Benton-Drones-Lead-Ingest.git
cd Benton-Drones-Lead-Ingest

# Optional: install Python dependencies (for true PDF export)
pip install -r requirements.txt

# Initialize the SQLite database
make init-db

# Start the server
ADMIN_PASSWORD=choose-a-strong-password make run
```

Then open these URLs in a browser:

```
http://127.0.0.1:8000/              → redirects to training guide
http://127.0.0.1:8000/signup        → public signup form
http://127.0.0.1:8000/admin-login   → admin login
http://127.0.0.1:8000/admin         → admin dashboard
```

### Testing end-to-end locally

1. Open `/signup` and submit a test lead with consent and typed-name signature.
2. Go to `/admin-login` and log in with the `ADMIN_PASSWORD` you chose.
3. View `/admin` to see the lead on the map and in the recent leads table.
4. Click **View** to open the lead detail page.
5. Open `/admin/lead/<id>/print` to see the printable consent form.
6. Download CSV, GeoJSON, or KML exports from `/admin`.

### Running tests

```bash
make test
```

## Security note

The local server is intended for development and trusted-network testing. Before exposing it to the internet:

1. Replace the placeholder waiver text in `lead_ingest/models.py` with legally reviewed language.
2. Set strong `ADMIN_PASSWORD`, `ADMIN_SESSION_SECRET`, and `CSRF_SECRET`.
3. Run behind HTTPS with a reverse proxy (see `deploy/Caddyfile` and `deploy/nginx.conf`).
4. See `docs/production-readiness-review.md` for the full production checklist.
